# 疼痛问诊小程序 — 使用教程

## 一、准备工作

### 1. 安装微信开发者工具

访问微信官方开发者平台下载对应版本：
- Windows 用户：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
- 选择"稳定版"下载安装即可

安装完成后，用微信扫码登录开发者工具。

### 2. 确认项目文件完整

你选择的文件夹中应包含以下文件和目录：

```
疼痛问诊小程序/
├── app.js                 # 小程序入口逻辑
├── app.json               # 全局配置（页面路由、导航栏等）
├── app.wxss               # 全局样式
├── project.config.json    # 项目配置
├── sitemap.json           # 搜索配置
├── README.md              # 项目说明
├── 教程.md                # 本教程
├── pages/                 # 页面目录
│   ├── home/              # 首页
│   ├── consult/           # 问诊采集页
│   ├── result/            # 分诊结果页
│   ├── records/           # 健康档案页
│   └── stats/             # 统计页
└── utils/
    └── triage.js          # 疼痛分诊规则模块
```

如果缺少任何文件，请重新获取完整项目包。

## 二、导入项目

### 步骤 1：打开微信开发者工具

双击桌面上的"微信开发者工具"图标启动。

### 步骤 2：导入项目

1. 点击左侧菜单的"小程序"选项卡
2. 点击右上角的"导入"按钮（或"+"号）
3. 在弹出的文件选择器中，导航到你保存项目的文件夹
4. 选择该文件夹后，工具会自动识别 `project.config.json`

### 步骤 3：填写项目信息

在导入对话框中：
- **AppID**：选择"测试号"（无需申请正式 AppID）
- **项目名称**：可以自定义，例如"疼痛问诊系统"
- 勾选"在当前目录中创建 project.config.json"（如果提示）

点击"确定"完成导入。

---

## 三、编译与预览

### 首次编译

导入成功后，开发者工具会自动编译项目。你会看到：
- 左侧是模拟器（显示小程序运行效果）
- 右侧是编辑器（显示代码文件）
- 下方是调试器（显示控制台输出）

如果编译成功，模拟器中会显示小程序首页，包含：
- 页面标题"疼痛智能问诊"
- "开始疼痛问诊"按钮
- "查看健康档案"按钮
- "后台统计"按钮

### 常见编译问题

| 问题 | 解决方法 |
|------|---------|
| 提示"未找到 app.json" | 确认导入的是项目根目录，而不是上级文件夹 |
| 页面空白 | 检查模拟器底部是否选择了正确的页面路径 |
| 样式错乱 | 尝试点击菜单栏"工具"→"刷新"或按 Ctrl+R |

## 四、功能体验流程

### 1. 开始问诊（患者端）

1. 在模拟器中点击首页的"开始疼痛问诊"按钮
2. 进入问诊采集页面，按提示填写：
   - **疼痛位置**：点击选择（头/胸/腹/腰背/四肢关节/全身）
   - **疼痛性质**：点击选择（刺痛/绞痛/烧灼痛/撕裂痛/钝痛/酸痛）
   - **疼痛时长**：点击选择（突发几小时/持续数日/慢性反复发作）
   - **诱发/缓解因素**：点击选择
   - **伴随症状**：多选（胸闷/大汗/呕血/头晕/肢体麻木/发热）
   - **文字描述**：输入疼痛的详细描述（或点击"语音输入示例"体验）
3. 点击"提交问诊信息"

### 2. 查看分诊结果

提交后自动跳转到结果页面，显示：
- **AI 结构化摘要**：整理后的问诊要点
- **疼痛部位分类**：如"胸痛"
- **疼痛性质分类**：如"撕裂痛"
- **风险等级**：Ⅰ级（极高危）至Ⅳ级（慢性慢病）
- **分诊建议**：如"立即急诊抢救"或"常规排队"
- **推荐科室**：如"急诊科"或"全科"

### 3. 查看健康档案

1. 返回首页，点击"查看健康档案"
2. 页面显示历次问诊记录列表
3. 每条记录包含：日期、疼痛部位、风险等级、分诊建议
4. 点击单条记录可查看详情

### 4. 查看后台统计

1. 返回首页，点击"后台统计"
2. 页面显示：
   - 疼痛部位分布（柱状图）
   - 风险等级占比（饼图）
   - 各科室分流数量（柱状图）
3. 数据基于本地存储的问诊记录自动生成

## 五、模拟不同风险等级

为了体验完整的分诊功能，可以尝试以下典型症状组合：

| 风险等级 | 症状组合示例 | 预期结果 |
|---------|------------|---------|
| Ⅰ级（极高危） | 位置：胸 + 性质：撕裂痛 + 症状：大汗 | 立即急诊抢救 |

| Ⅲ级（普通急症） | 位置：四肢关节 + 性质：钝痛 + 时长：突发几小时 | 当日全科优先 |
| Ⅳ级（慢性慢病） | 位置：腰背部 + 性质：酸痛 + 时长：慢性反复发作 | 常规排队，慢病管理 |

## 六、项目结构说明

### 页面文件对应关系

| 页面 | 功能 | 文件位置 |
|-----|------|---------|
| 首页 | 导航入口 | `pages/home/` |
| 问诊采集 | 患者填写疼痛信息 | `pages/consult/` |
| 分诊结果 | 显示 AI 分析结果 | `pages/result/` |
| 健康档案 | 历史问诊记录 | `pages/records/` |
| 后台统计 | 数据可视化展示 | `pages/stats/` |

### 核心逻辑文件

- `utils/triage.js`：疼痛分诊规则模块，根据症状组合判断风险等级
- `app.js`：小程序生命周期管理，全局数据存储

## 七、后续扩展建议

当前版本为前端演示原型，核心 AI 功能使用本地规则模拟。后续可接入真实服务：

1. **接入 FastChat**：将 `consult.js` 中的文字描述提交到 FastChat 服务，实现自然语言问诊
2. **接入 MONAI 模型**：将 `triage.js` 替换为后端 MONAI 分类模型 API 调用
3. **接入 OpenEMR**：将档案数据同步到 OpenEMR 电子病历系统

## 八、常见问题

**Q：为什么语音输入是"示例"而不是真实录音？**
A：当前版本为演示原型，语音功能需要接入微信录音 API 和语音识别服务。点击"语音输入示例"会自动填入一段模拟语音转文字内容。

**Q：数据保存在哪里？**
A：当前使用微信小程序本地存储（`wx.setStorageSync`），数据保存在当前设备上。如需多端同步，需接入后端数据库。

**Q：可以发布为正式小程序吗？**
A：可以，但需要：
1. 申请微信小程序正式 AppID
2. 完善隐私协议和用户授权
3. 接入后端服务替代本地模拟逻辑
4. 通过微信审核
