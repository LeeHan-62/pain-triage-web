# 疼痛问诊系统模型接入说明

当前项目已新增 `api-server` 后端接入层，用于统一接入 FastChat、MONAI、OpenEMR 和 KoBERT-Emotion。网页版仍然可以直接打开；如果后端服务已启动，问诊提交会优先调用后端模型接口，如果后端不可用，会自动回退到浏览器本地规则。

## 一、目录结构

```text
api-server/
├── main.py                    # FastAPI 后端入口
├── requirements.txt           # Python 依赖
├── .env.example               # 环境变量示例
├── adapters/
│   ├── fastchat_client.py     # FastChat 问诊文本结构化
│   ├── monai_client.py        # MONAI 多分类模型调用
│   ├── emotion_client.py      # KoBERT-Emotion 情绪强度分类
│   ├── openemr_client.py      # OpenEMR 患者档案写入
│   └── mock_rules.py          # 本地模拟规则
└── models/
    └── schemas.py             # 请求与响应数据结构
```

## 二、启动后端

在 `api-server` 目录下执行：

```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

启动成功后访问：

```text
http://127.0.0.1:8000/api/health
```

如果返回 `status: ok`，说明后端接入层已经启动。

## 三、接入模式

`.env` 中的 `APP_MODE` 控制模型调用方式：

```text
APP_MODE=mock
```

可选值：

| 模式 | 含义 |
|---|---|
| `mock` | 不调用真实模型，只使用本地规则，适合演示 |
| `hybrid` | 优先调用真实模型，失败后回退本地规则 |
| `real` | 按真实模型接口调用，仍会记录失败原因 |

建议先使用 `mock` 跑通网站，再切换到 `hybrid` 逐个接入真实服务。

## 四、FastChat 接入

FastChat 用于把患者自由文本整理为结构化问诊字段。当前后端按 OpenAI 兼容接口调用：

```text
FASTCHAT_BASE_URL=http://127.0.0.1:8001/v1
FASTCHAT_API_KEY=EMPTY
FASTCHAT_MODEL=clinical-chat
```

后端会调用：

```text
POST /v1/chat/completions
```

期望输出 JSON 字段：

```json
{
  "location": "胸",
  "nature": "撕裂痛",
  "duration": "突发几小时",
  "factor": "休息不缓解",
  "symptoms": ["大汗", "胸闷"],
  "chief_complaint": "胸部撕裂样疼痛伴大汗",
  "red_flags": ["胸痛", "大汗"]
}
```

## 五、MONAI 接入

MONAI 用于疼痛部位、性质和风险等级多分类。当前后端会向配置地址发送：

```text
MONAI_API_URL=http://127.0.0.1:8002/predict
```

请求体包含：

```json
{
  "text": "患者原始描述",
  "structured": {},
  "task": "pain_triage_multiclass",
  "labels": {
    "location": ["头痛", "颈肩痛", "胸痛"],
    "nature": ["撕裂痛", "绞痛", "烧灼痛"],
    "risk": ["Ⅰ级", "Ⅱ级", "Ⅲ级", "Ⅳ级"]
  }
}
```

建议 MONAI 服务返回：

```json
{
  "location": "胸痛",
  "nature": "撕裂痛",
  "risk": "Ⅰ级",
  "confidence": 0.93
}
```

## 六、KoBERT-Emotion 接入

KoBERT-Emotion 用于识别患者口述中的痛苦情绪强度：

```text
KOBERT_EMOTION_API_URL=http://127.0.0.1:8003/predict
```

建议返回：

```json
{
  "label": "重度痛苦",
  "score": 0.91
}
```

结果会显示在分诊结果页的“患者痛苦情绪强度”字段中。

## 七、OpenEMR 接入

OpenEMR 用于写入患者档案和慢病记录：

```text
OPENEMR_BASE_URL=http://127.0.0.1:8081/apis/default
OPENEMR_ACCESS_TOKEN=你的访问令牌
OPENEMR_PATIENT_ID=默认患者ID
```

当前逻辑会先创建一次 encounter，再写入一条临床记录。不同 OpenEMR 部署版本的接口字段可能不同，如果你的 OpenEMR 接口格式不一致，只需要修改：

```text
api-server/adapters/openemr_client.py
```

## 八、前端调用方式

网页版新增了：

```text
web-version/js/api.js
```

问诊提交时会调用：

```text
POST http://127.0.0.1:8000/api/triage
```

如果后端未启动，页面不会中断，会自动使用 `web-version/js/triage.js` 的本地规则完成分诊。

## 九、上线注意事项

如果要部署到互联网，静态网页和后端 API 要分别部署：

| 部分 | 建议部署方式 |
|---|---|
| `web-version` | GitHub Pages、Vercel、Netlify、对象存储静态网站 |
| `api-server` | 云服务器、Render、Railway、腾讯云、阿里云 |
| FastChat / MONAI / KoBERT | GPU 服务器或内网模型服务 |
| OpenEMR | 医院内网或受控云环境 |

上线时需要把 `web-version/js/api.js` 中的 `API_BASE_URL` 改成公网后端地址，例如：

```javascript
const API_BASE_URL = 'https://你的后端域名';
```

医疗数据涉及隐私，正式使用前应加入登录鉴权、HTTPS、访问审计、患者授权和数据脱敏。
