# 疼痛智能问诊与分诊微信小程序原型

这是一个面向社区门诊、全科诊室和预检分诊场景的微信小程序原型。小程序实现患者自助疼痛问诊、结构化摘要生成、疼痛部位分类、疼痛性质分类、风险分级、自动分流、档案记录和后台统计。

## 已实现功能

- 患者自助问诊：采集疼痛位置、性质、时长、诱发缓解因素和伴随症状。
- 语音输入占位：当前用“填入语音识别示例”模拟语音转文字结果。
- AI 分类占位：`utils/triage.js` 使用本地规则模拟 FastChat 与 MONAI 分类结果。
- 风险分级：支持Ⅰ级、Ⅱ级、Ⅲ级、Ⅳ级疼痛风险判断。
- 自动分诊：根据风险等级推荐急诊抢救区、优先诊室或常规专科队列。
- 档案联动占位：将结果保存到小程序本地缓存，后续可改为写入 OpenEMR。
- 后台统计：统计疼痛部位、疼痛性质、风险等级和高危患者数量。

## 页面结构

- `pages/home/home`：系统首页与流程说明。
- `pages/consult/consult`：患者自助疼痛问诊页面。
- `pages/result/result`：AI 分诊结果页面。
- `pages/records/records`：疼痛健康档案页面。
- `pages/stats/stats`：后台统计页面。
- `utils/triage.js`：本地疼痛分类与风险分级规则。

## 运行方式

1. 打开微信开发者工具。
2. 选择“导入项目”。
3. 项目目录选择本文件所在文件夹。
4. AppID 可选择测试号或使用你自己的小程序 AppID。
5. 编译运行后，从首页点击“开始疼痛问诊”。

## 后续接入建议

- 将 `utils/triage.js` 中的本地规则替换为后端接口请求。
- FastChat 服务负责多轮问诊、语音文本整理和结构化字段抽取。
- MONAI 服务负责多任务分类，输出部位分类、性质分类和风险分级。
- OpenEMR 接口负责写入患者健康档案和慢病随访提醒。
- 后台统计可改为由服务端按月聚合，支持导出科研数据。
